## Skill Format

Every skill must include:

1. **Metadata**: Category, Scope, Difficulty, Last Updated, Effectiveness
2. **Problem**: What does this solve?
3. **Solution Overview**: High-level approach
4. **Implementation**: 
   - Files to Create table
   - Code patterns for each supported stack
5. **Key Principles**: Stack-agnostic rules
6. **Stack Variations**: Differences per technology
7. **Integration**: How it fits with other skills
8. **Common Mistakes**: What to avoid
9. **Validation Checklist**: How to verify it's working

## Adding New Skills

### For New Tech Stacks

1. Create directory: `skills-library/{domain}/{stack}/`
2. Document stack-specific patterns
3. Update this index
4. Link to tech-agnostic parent skill if applicable

### For Tech-Agnostic Patterns

1. Create in appropriate domain folder
2. Provide implementations for 2+ stacks
3. Mark as "Universal" in metadata
4. Reference from stack-specific skills
